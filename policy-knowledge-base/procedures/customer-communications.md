# Customer communications

Customer communication policies cover who teams can speak with, which channels are approved, how often outreach can occur, and what language should be avoided.

## Communication categories

| Category | Example policy area | Demo treatment |
| --- | --- | --- |
| Authorized users | Spousal authorization | Expandable rules and required documentation |
| Account information | Updating account information | Stepper for verification and system update |
| Phone communication | Mini Miranda, call frequency, call recording | Hints for compliance-critical reminders |
| Paper communication | Approved correspondence, signatures | Document checklist |
| Text and email | Channel criteria and templates | Tabs by channel |
| Prohibited communication | Common pitfalls and phrases | Warning callouts |

## Update account information

{% stepper %}
{% step %}
### Verify requester authority

Confirm the requester is the customer or an authorized user before making changes.
{% endstep %}

{% step %}
### Capture required documentation

Attach the required document or consent record to the account before updating policy-controlled fields.
{% endstep %}

{% step %}
### Update the servicing system

Make the update in the approved source system and record the policy reference used.
{% endstep %}

{% step %}
### Confirm communication preference

Check whether any cease and desist, channel restriction, or accessibility preference applies.
{% endstep %}
{% endstepper %}

{% hint style="danger" icon="ban" %}
Prohibited communication guidance should be short, direct, and easy to find from search. Long lists of risky phrases can live inside expandable blocks.
{% endhint %}
