# Policy operating model

Arivo can manage many policies by turning each document section into a structured page with ownership, approval dates, and review status.

| Control | Example |
| --- | --- |
| Owner | Account Servicing Department |
| Reviewer | Compliance Operations |
| Source system | Norman LMS |
| Review cycle | Quarterly |
| Change vehicle | GitBook change request or policy owner update |

{% hint style="info" icon="shield-check" %}
Policy pages should be short enough to scan but detailed enough to answer the operational question without opening a separate document.
{% endhint %}

## Scalable content pattern

- One section homepage for every major policy area.
- One page for every meaningful policy or procedure.
- Metadata at the top of each page.
- Hints for warnings, notes, and compliance-sensitive reminders.
- Expandable sections for exceptions and edge cases.
- Changelog entries for material updates.
